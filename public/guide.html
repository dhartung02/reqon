Reqon is your self-hosted home base for the job search. It tracks every job by match strength, finds new roles from board APIs, and helps you draft and apply. This guide explains the scores, statuses, what needs you today, and the moving parts behind Settings.

Scores & match strength

Fit = domain / résumé match (a condensed version of the 7-axis model). Interview probability = the estimated odds of landing a screen given seniority, remote competition, and how directly your background maps — kept deliberately separate from fit, because some perfect-fit roles are genuinely hard to land. Expected value = fit × probability ÷ 10 — your best single number for allocating apply effort.

Match strength (Strong / Possible / Long shot) is set by expected value, not raw fit. Bands (shown on an expanded card):

BandScoreMeaning
Excellent8.0+Strong territory — apply with a tailored résumé
Strong6.5–7.9Strong / Possible — strong apply
Possible5.0–6.4Possible — apply if capacity allows
Low priority<5.0Long shot — skip unless something else pulls you in

All scores are estimates and salary ranges are indicative — verify on each board.

Link confidence

BadgeWhat it means
VerifiedA live req was confirmed this session. Safe to apply.
Board-onlyThe link goes to the company's careers board, not a specific req. Individual req URLs rotate and expire — filter for the title when you land.
UnverifiedSourced from an uploaded doc, not yet confirmed. Check before relying on it.

What "Needs verification" asks of you: open the link and confirm the posting is still live, then change confidence to Verified (or archive it if it's dead). It's a manual confidence check — there's no automated step — so you don't spend effort applying to a posting that's already closed.

Status & match-strength edge

Each row carries a status pill — applied green, in-interview teal (solid at offer), rejected red (row dimmed), archived faded. The thin left edge marks match strength: Strong emerald · Possible amber · Long shot muted.

Ways to apply

This tells you how a posting can be filled. It's inferred from the source (the ATS the posting lives on) and can be overridden per row.

ModeMeaning
fillableA public ATS (Greenhouse, Ashby, Lever, Workable, SmartRecruiters, Recruitee, Personio, Teamtailor) — the apply-assist can pre-fill the form.
gatedA login-walled ATS (Workday, iCIMS, Taleo, SuccessFactors, BambooHR, Phenom, Jobvite) — open it and apply manually, or use Simplify Copilot.
simplifyBest handled by the Simplify Copilot browser extension.
manualApply by hand (LinkedIn, custom portals, quick-adds).

Sources & source health

Reqon polls public board APIs for the companies on your watchlist. In Settings → What you’re looking for you can enable/disable each source and add a company by pasting a careers URL (the detector recognizes the ATS automatically).

Source health reports, per source, the result of the most recent check:

NumberWhat it tells you
MatchesThe number that matters — roles that cleared your fit / match-strength bar and were added.
ErrorsParse/API failures last run. Anything > 0 means that source needs attention.
Companies / PostingsHow many companies were polled and how many raw postings were scanned — process telemetry, useful when debugging a quiet source.

Data safety & deleted rows

Every board save snapshots the previous data first, and corruption-shaped saves (an empty save over a full store, or one that drops a large share of rows) are refused.

Deleted rows are recoverable. When you delete a row it isn't erased — it's hidden and marked deleted, so the deletion still syncs to your other devices instead of the row reappearing. "Purge" permanently removes those hidden rows (after a snapshot). (Internally they're kept as a soft-delete marker so the deletion syncs across devices instead of the row reappearing.)

Integrations

Job aggregators — TheirStack & Apify (Fantastic.jobs)

These are optional paid third-party APIs that reach the gated ATSes (Workday, iCIMS, etc.) Reqon can't poll directly. Each needs an API key (free trial plans exist; real volume is paid). Add keys under Settings → Account & data. Both stay inert until a key is present — they never block a check or digest run.

iOS push (APNs)

Push to the iOS app uses token-based Apple Push Notification service, which requires an Apple Developer account. You provide:

FieldWhere it comes from
.p8 keyApple Developer → Keys → create an APNs key and download the .p8 file once. It's the private key the server uses to sign push tokens (the modern replacement for push certificates).
Key IDShown next to that key in Apple Developer → Keys.
Team IDApple Developer → Membership.
Bundle IDYour app's identifier, e.g. com.reqon.app.
Environmentsandbox for dev / Expo Go / TestFlight builds; production for App Store builds.

Remote access (HTTPS)

iOS blocks plaintext HTTP, so connecting from outside your LAN needs HTTPS via Tailscale Funnel or Caddy. Set your public hostname under Settings → Account & data → Remote access URL and the pairing QR will use it.
