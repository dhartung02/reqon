Link confidence. Verified = I confirmed a live req this session. Board-only = link goes to the company's careers board (filter for the title); individual req URLs rotate/expire. Unverified = sourced from your uploaded doc, not yet confirmed — check before relying on it.
Fit = domain/résumé match (your earlier 7-axis model, condensed). Interview probability = my estimate of landing a screen given seniority, remote competition, and how directly your background maps — deliberately separate from fit, since some perfect-fit roles are hard to land. Expected value = fit × probability, your best apply-effort allocator. Tier A refined by expected value, not raw fit. All scores are estimates, salary ranges indicative — verify on each board.
Status colors. Cards you've applied to are tinted green; in-interview stages teal (brightest at offer); rejected dimmed red; archived faded. Left edge = tier, right edge = status. Untinted = not yet addressed.

Lifecycle tabs

Comma-separated statuses routed to each board tab. Valid: Not Applied, Applied, Recruiter Screen, Hiring Manager, Panel, Offer, Rejected, Archived. Any unlisted status falls into Open.

Hygiene lanes

Lanes are non-destructive: closed reqs are surfaced for review and archived only on confirm (a snapshot is taken first) — never auto-deleted. The needs-verify lane always includes unverified links and leads.

AI application assistant

Per-req "✍ AI draft" composes a cover note or screening answer from your profile + narratives + JD. Output is editable and never auto-submitted.

Morning digest

The always-on server composes from your data and delivers at the set time. A file fallback is always written to agent/digest-latest.html. MCP connectors aren't reachable from the server — use a Slack webhook or SMTP. Save settings before sending a test.

Data safety

Every board save snapshots the previous data first; only the newest 50 auto-snapshots are kept (manual & phase snapshots are never auto-deleted). A save that would drop more than the guard % of rows — or an empty save over a non-empty store — is refused as likely corruption. Set the guard to 0 to allow any size drop.
